## 2. 이 서비스가 뭔가요

**취준생의 만성 불안·고립감**을 24시간 곁에서 들어주는 **AI 동반자(챗봇)** 서비스입니다.

핵심 아이디어(딥 인터뷰로 확정):
> "설문·리서치가 아니라 **실제로 도움이 되는 대화**를 제공하고, 그 대화에서 **자연스럽게 '어려움 데이터'가 만들어져** 나중에 관련 기관과의 협업·맞춤 매칭에 쓰인다."

- **사용자 관점:** 힘들 때 언제든 대화 → 위로받고 루틴 회복 → 나에게 맞는 공개 기관/정책 정보 추천
- **서비스 관점:** 대화에서 구조화된 어려움 데이터 축적(이게 성공 지표 = North-star) → (2차) 실제 기관 연결

**MVP 범위:** AI 대화 + 어려움 데이터 축적 + 큐레이션된 공개기관 추천까지.
**2차로 미룬 것:** 실제 기관 제휴·실명 연결, 취업기회(채용) 기능.

## 5. 사용법 — 실행하기

### A. 처음 한 번만 (환경 준비)
1. **열쇠 2개** — 이미 `.env.local`에 채워져 있음: Supabase 주소 + Anthropic(Claude) API 키.
2. **데이터베이스 설정** — 이미 완료(Supabase SQL Editor에 `supabase/SETUP_ALL.sql` 실행함).
3. **이메일 인증 끄기 (중요!)** — Supabase → `Authentication` → `Sign In / Providers` → `Email` → **`Confirm email` OFF** → 저장.

### B. 매번 실행
```bash
cd /Users/yugdong-yeon/workspace/vibecoding2
npm run dev        # 개발 서버 (http://localhost:3000)
```
브라우저에서 **http://localhost:3000** 접속.

### C. 써보는 순서
1. **회원가입** (이메일 + 비밀번호 + 별명 / 실명 안 넣음)
   - ⚠️ 이메일 인증을 껐다면 **새 이메일**로 가입하세요(이전 미인증 계정은 로그인 안 됨).
2. **연령 확인** → "만 19세 이상입니다"
3. **동의** → AI(Claude, 미국) 처리 안내에 동의 (이거 없으면 대화 안 열림 = 법적 필수)
4. **대화** → AI 동반자와 이야기
5. **내 기록**(`/journal`) · **추천**(`/recommendations`) 확인
6. (테스트) 위기 관련 표현 → **핫라인 카드**가 뜨는지 확인

### D. 유용한 명령어
```bash
npm run build      # 배포용 빌드
npm test           # 유닛 테스트 (75개)
npm run typecheck  # 타입 검사
```

## 7. 안전·개인정보 원칙
> ⚠️ 이 원칙들은 설계 합의에서 정한 **비협상 규칙**입니다. 코드 수정 시 절대 약화 금지.

- **위기 대응:** 모든 사용자 메시지를 답변 전에 검사 → 위기 시 핫라인(109/1577-0199) 즉시 표시. 검사 실패해도 **핫라인을 보여주는 쪽으로 안전하게 실패**(정상 응답으로 넘어가지 않음). MVP엔 사람 상담원 없음 — 핫라인이 최종 안내.
- **국외이전 동의:** 대화는 첫 메시지부터 Claude(미국)로 전송됨 → **대화 전에 반드시 별도 동의**(법적 필수). `/chat` 화면과 `/api/chat` **양쪽에서** 강제.
- **가명 우선:** 기본은 별명만. 실명은 기관연결 동의가 있을 때만 → **데이터베이스 트리거로 강제 차단**.
- **동의 원장:** 동의는 기록만 추가(append-only), 철회 가능. 계정삭제 시 대부분 삭제하되 동의·위기 기록은 법적 보존 위해 **익명화**.
- **관리자 통계:** 개인 식별 불가, 집계 숫자만. 관리자 이메일 허용목록으로만 접근.

## 8. 기술 구조 요약

```
[사용자] → 반응형 웹/PWA (Next.js 16)
             │
             ├─ 회원가입/동의 게이트 (Supabase Auth + proxy)
             ├─ /chat → AI 대화 (Claude Opus) + 입력측 위기감지 (Claude Haiku)
             │            └→ 대화 저장 (Postgres, RLS 보호)
             │                  └→ (백그라운드) 어려움 추출 (Edge Function) → difficulty_data
             ├─ /recommendations → 규칙기반 큐레이션 → 공개기관 추천
             ├─ /journal → 내 대화·루틴 기록
             └─ /admin/metrics → 집계 통계 (관리자 전용)
```
- **프론트+백:** Next.js 16 (App Router, TypeScript), Tailwind v4, PWA
- **데이터/인증:** Supabase (Postgres + Auth + RLS + Edge Functions)
- **AI:** Claude Opus(대화) / Claude Haiku(위기감지·추출)
- **테스트:** Vitest(유닛) · pgTAP(DB) · Playwright(E2E)
